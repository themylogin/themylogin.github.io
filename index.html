View more

My main role in this project is backend development. The following stack is used:

  • python + uwsgi + Flask as RESTful API server
  • aiohttp for handling 1000+ online video players
  • celery + RabbitMQ as asynchronous, background and periodic tasks queue
  • PostgreSQL as persistent database
  • Redis as database buffer and inter-process communicator
  • Elasticsearch + Logstash + Kibana for telemetry collection and analysis
  • AWS S3 + AWS CloudFront for storing and serving media files
  • Docker + docker-compose for small deployments and AWS ECS for high-availability deployments (also supported by ELB and RDS)

Besides servers, I've also developed hardware video player:

  • i.MX 6-based ARM computer (Hummingboard)
  • Debian Jessie
  • Video player software is written on python with several C modules where extra performance is needed. Supports synchronous playback on multiple devices («video wall»)
  • gtk3 UI
  • gstreamer as multimedia framework
  • Aambient light sensor, RFID reader, GPS, GPIO buttons, HDMI-cec control
  • Secure Boot protection, hardware-encrypted root volume, SD card wearout protection techniques, rescue boot mode

And B2B TV video player application:

  • Written with modern Javascript (ES6)
